## 📚 学习方法建议

### 1. 边学边练
- **不要只看不练**：每学一个概念立即编写代码验证
- **从简单开始**：先写简单的脚本，逐步增加复杂度
- **解决实际问题**：结合工作中的实际需求编写脚本

### 2. 调试技巧
```bash
# 开启调试模式
set -x          # 显示执行的命令
set -e          # 遇到错误立即退出
set -u          # 使用未定义变量时报错
set -o pipefail # 管道中任何命令失败都返回失败

# 组合使用（推荐）
set -euxo pipefail
```

### 3. 查阅帮助
```bash
# 查看命令帮助
man command
command --help
info command

# 在线资源
# https://www.shellcheck.net/ - 脚本检查工具
# https://explainshell.com/ - 命令解释工具
```

### 4. 最佳实践
- **使用有意义的变量名**
- **添加注释说明复杂逻辑**
- **进行错误处理**
- **使用函数避免重复代码**
- **遵循一致的代码风格**

## 🚨 常见错误避免

### 1. 语法错误
```bash
# 错误：等号两边有空格
name = "value"

# 正确：等号两边无空格
name="value"
```

### 2. 变量使用错误
```bash
# 错误：未加引号
if [ $var == "test" ]; then

# 正确：加双引号
if [[ "$var" == "test" ]]; then
```

### 3. 路径问题
```bash
# 错误：未处理包含空格的路径
for file in $(ls *.txt); do

# 正确：使用数组
mapfile -t files < <(find . -name "*.txt")
for file in "${files[@]}"; do
```

### 4. 权限问题
```bash
# 记得给脚本执行权限
chmod +x script.sh

# 检查脚本语法
bash -n script.sh
```
